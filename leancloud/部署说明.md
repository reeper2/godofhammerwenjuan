# LeanCloud 国内版 — 问卷数据收集

[LeanCloud 国内版](https://console.leancloud.cn) 服务器在国内，**无需访问 Google**，适合大陆受访者和管理员使用。

免费开发版额度对个人/课题问卷通常够用。

## 1. 注册并创建应用

1. 打开 https://console.leancloud.cn 注册登录  
2. **创建应用** → 名称如「AI数字化转型问卷」  
3. 进入应用 → **设置 → 应用凭证**，记下：
   - **App ID**
   - **App Key**（填进网站 `config.js`，仅用于提交答卷）
   - **Master Key**（仅保存在本机 `.env`，用于导出数据，**不要**写进网页或提交到 GitHub）

4. 记下 **API 域名**（形如 `https://xxxx.api.lncld.net`）

## 2. 创建数据表（Class）

1. **数据存储 → 结构化数据 → 创建 Class**  
2. 名称：`SurveyResponse`（与 `config.js` 中 `className` 一致）  
3. 不需要手动建字段，首次提交会自动生成列

## 3. 设置权限（重要）

进入 Class `SurveyResponse` → **权限**：

| 操作 | 建议设置 |
|------|----------|
| `add`（新建） | 所有用户 ✓ |
| `find`（查询） | 所有用户 ✗ |
| `update` / `delete` | 所有用户 ✗ |

这样：访客只能提交，**不能**在网页上批量读取他人答卷；你通过控制台或本机脚本用 Master Key 导出。

## 4. 填写网站配置

编辑项目根目录 `config.js`：

```javascript
window.SURVEY_CONFIG = {
  leancloud: {
    appId: '粘贴AppID',
    appKey: '粘贴AppKey',
    apiServer: 'https://xxxx.api.lncld.net',
    className: 'SurveyResponse'
  },
  consoleUrl: 'https://console.leancloud.cn/apps/你的应用ID/storage/overview'
};
```

保存后 `git push`，GitHub Pages 约 1–2 分钟生效。

## 5. 本机导出密钥（仅管理员电脑）

```bash
cp .env.example .env
# 编辑 .env，填入 App ID、Master Key、API 域名
```

导出全部答卷：

```bash
python3 scripts/fetch_submissions.py
python3 scripts/export_report.py output/survey_results.json -o output/汇总报告.md
```

## 6. 在控制台查看

**数据存储 → SurveyResponse** 可实时看到每条记录；`payload` 列为完整 JSON。

可点击 **导出** 为 JSON/CSV，再在 Excel 中做透视和图表。

## 常见问题

**Q：App Key 放在网页里安全吗？**  
已限制为仅能 `add` 不能 `find`，风险可控。Master Key 切勿写入 `config.js` 或公开仓库。

**Q：GitHub Pages 国内慢怎么办？**  
可同时把同一文件夹部署到 **Gitee Pages** 或学校服务器，仍使用同一套 LeanCloud 配置即可。

**Q：额度不够？**  
可升级套餐，或改用腾讯云开发 CloudBase（需自行改提交接口，逻辑类似）。
